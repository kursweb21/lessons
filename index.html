Система контроля версий GIT

Список команд для GIT

  • git init - подключает GIT к рабочей папке
  • git pull - работает как комбинация команд git fetch и git merge, т.е. Git вначале забирает изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой.
  • git branch - это своего рода “менеджер веток”. Она умеет перечислять ваши ветки, создавать новые, удалять и переименовывать их.
  • git chekout - используется для переключения веток и выгрузки их содержимого в рабочую директорию
  • git merge - используется для слияния одной или нескольких веток в текущую.
  • git fetch - связывается с удалённым репозиторием и забирает из него все изменения, которых у вас пока нет и сохраняет их локально.
  • git push - используется для установления связи с удалённым репозиторием, вычисления локальных изменений отсутствующих в нём, и собственно их передачи в вышеупомянутый репозиторий
  • git add добавляет содержимое рабочей директории в индекс (staging area) для последующего коммита
  • git commit - создает точку сохранения и сообщение к ней
  • git status - посмотреть статусы
